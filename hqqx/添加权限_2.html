“Set Time Range”

1、此处的逻辑为“修改”逻辑，将权限生效时间修改到某一个时间区间内实行


2、增加时间区间选择器，精准天，并且可以清空所选天数


3、点击【Sure】提示“确认修改”页面，

A:Set Time Range的时间区间与现有的时间区间无重叠的情况下，无提示。例：今天是1月1日，我的权限到期日是1月5日，本次添加的时间区间是1.6日到1月10日，那么将没有提示

B：如果Set Time Range的时间与现有的时间区间有重叠，那么将会显示“确认修改”页面，文案为“Confirm that the xxxx permission execution time of xxx will be changed to xxxx-xx-xx to xxxx-xx-xx”，例：今天是1月1日，我的权限到期日是1月5日，本次添加的时间区间是1.2日到1月10日，那么将提示上文文案

C:如果现有的时间包含Set Time Range的时间，那么将提示“无法指定此时间区间”的提示窗口，文案为“Because the existing permission time includes the time within this Set Time Range, there is no need to add permissions for this time period for the time being!”，例：今天是1月1日，我的权限到期日是1月5日，本次添加的时间区间是1.2日到1月3日，那么将提示上文文案

D:该用户此时无该权限，为用户添加权限时，无任何提示
